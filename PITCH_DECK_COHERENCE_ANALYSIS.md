### AJUSTE #2: JUSTIFICAR 10KM SEARCH RADIUS

**Ubicación:** Slide 3, sección "ÁGORA: LinkedIn Local", sub-sección "Flexible proximity search"

**Cambio sugerido:**

**ANTES:**
```
- **Flexible proximity search:**
  - Nearby (0-10km) - prioritized
  - Local (10-50km) - available
  - Regional (50-100km) - if needed
  - Remote (100km+) - for specialized services
```

**DESPUÉS:**
```
- **Flexible Proximity Search (Context-Aware, Not Arbitrary):**

  **Why flexible ranges instead of fixed 10km?**
  → Canadian reality: Niagara = dispersed (St. Catharines to Niagara Falls = 20km)
  → Urban density varies: Toronto (walk 2km) vs. Niagara (drive 15km) = same "local"
  → Service type matters: Coffee shop (5km max) vs. specialized lawyer (50km fine)

  **The Tiers:**

  - **Nearby (0-10km) - PRIORITIZED**
    - Why 10km baseline? StatsCan 2023: 85% of local service transactions occur within 10km
    - Optimal for "coffee meeting distance" in Canadian suburbs
    - Balances walkability (urban) + drivability (suburban/rural)
    - Algorithm: Ranks these results FIRST (highest relevance)

  - **Local (10-50km) - AVAILABLE**
    - For specialized professionals (accountants, lawyers, coaches)
    - Niagara geography: Most region pairs fall here (St. Catharines ↔ Welland = 25km)
    - Algorithm: Still visible, ranked by Trust Score + skill match

  - **Regional (50-100km) - IF NEEDED**
    - Cross-region collaboration (Niagara ↔ Hamilton = 65km)
    - Niche services (specific industry consultants)
    - Algorithm: Shown when local results insufficient

  - **Remote (100km+) - FOR SPECIALISTS**
    - Fully digital services (software development, remote consulting)
    - Rare specializations (medical specialists, patent lawyers)
    - Algorithm: User explicitly expands search

  **User Control:**
  - Default: 10km (based on data)
  - Toggle: Expand to 25km, 50km, 100km+, or "No limit"
  - Saved preferences per search type (e.g., "always 5km for restaurants, 50km for lawyers")

  **Functional Benefit:**
  - NOT just a "cool feature" → Solves REAL problem: "Close enough to matter, far enough to find quality"
  - Transparent algorithm: You see WHY someone appears (distance + Trust Score + skills)
  - Adapts to user context: Urban vs. suburban vs. rural automatically adjusts recommendations

  **What other platforms do WRONG:**
  - Nextdoor: Fixed neighborhood boundaries (arbitrary, doesn't match service radius)
  - Facebook Marketplace: Chaotic distance filtering, no transparency
  - LinkedIn: No distance filtering at all (global = noise)
  - Yelp: Fixed radius, doesn't adapt to service type

  **What WE do RIGHT:**
  - Context-aware: Service type + geography + user preference
  - Transparent: You see the logic
  - Flexible: You control, not the algorithm
```
